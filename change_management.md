# 变更管理协议

## 原则
**任何可能导致主人无法访问的变更，必须：**
1. 提前告知并获得确认
2. 准备备用访问方案
3. 提供详细恢复指引
4. 保留回滚能力

## 禁止行为
- ❌ 未经确认更改公网IP
- ❌ 未经确认更改端口
- ❌ 未经确认更改访问域名
- ❌ 未经确认更改认证方式

## 必须行为
- ✅ 变更前24小时通知
- ✅ 提供变更时间窗口选择
- ✅ 准备测试环境验证
- ✅ 创建恢复指引文档
- ✅ 保留至少一种备用访问方式

## 网络变更检查清单

### 变更前：
1. [ ] 确认主人当前访问方式
2. [ ] 准备至少2种备用访问方案
3. [ ] 生成《变更指引文档》
4. [ ] 测试新配置在测试环境
5. [ ] 获得主人书面确认

### 变更中：
1. [ ] 按步骤执行，每一步验证
2. [ ] 保持至少一种访问方式可用
3. [ ] 实时沟通变更进度

### 变更后：
1. [ ] 验证所有访问方式
2. [ ] 提供《访问方式汇总》
3. [ ] 监控24小时稳定性
4. [ ] 收集主人反馈

## 紧急恢复包
每次变更必须生成紧急恢复包，包含：
1. 当前访问方式汇总
2. 备用访问方式
3. 一键恢复脚本
4. 技术支持联系方式

## 失联应急预案
如果意外失联：
1. 主人执行：`./紧急恢复.sh`
2. 脚本自动尝试所有备用访问方式
3. 如30分钟内无法恢复，自动回滚
4. 发送短信/邮件通知主人
